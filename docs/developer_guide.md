# Developer Guide

## Principles

1. **Multiplayer-First** – design APIs assuming multiple simultaneous players.
2. **Every Function Tested** – add a unit test for each public function.
3. **Modularity** – isolate concerns (`net`, `game`, `ui`, `data`).
4. **Human-Friendly Layout** – keep directories shallow and descriptive.

## Workflow Checklist

1. Define a module's responsibility before coding.
2. Implement tests alongside code.
3. Run formatting, linting and tests prior to commits.
4. Document new modules in `docs/`.
